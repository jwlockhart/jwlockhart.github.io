When your ID numbers get rounded

Almost everyone doing computational social science will, at some point, encounter really big ID numbers. IDs for many data sets can stretch into the quintillions (that's a billion-billions). This happens, for example, in twitter data, where tweets have ID numbers like 1095603268586270720. Problem is, when we start working with this data, we often see it converted to scientific notation like this: \(1.09560326858627 Ã— 10^18\).
